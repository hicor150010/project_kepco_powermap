description: 수정 전 실제 렌더되는 컴포넌트 확인 필수. 중복 로직 퍼뜨리지 말고 한 곳에서 처리.
type: feedback
---

1. **수정 전 실제 렌더 파일 확인**: 같은 기능이 여러 컴포넌트에 있을 수 있다 (예: SearchPanel vs Sidebar 둘 다 검색창). 반드시 브라우저에서 실제로 쓰이는 컴포넌트를 먼저 특정.

2. **덕지덕지 금지**: 여러 이벤트 핸들러(onChange, onClick, onFocus)에 같은 로직 복붙하지 말 것. 함수 하나로 뽑거나, 한 곳에서만 처리.

3. **테스트 후 보고**: "빌드 성공"만으로 끝내지 말 것. 실제 동작 확인까지.

**Why:** 잘못된 파일을 수정해서 시간 낭비 + 여러 곳에 중복 로직이 퍼져서 유지보수 어려움.
**How to apply:** grep으로 placeholder/텍스트 검색해서 실제 렌더 파일 특정 → 한 곳만 수정 → 동작 확인.
