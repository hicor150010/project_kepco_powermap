## 목표

태양광 설치 잠재 타겟(유리온실, 축사, 대형 창고/건물)을 지도에 표시하고, 각 타겟 위치 기준으로 **근처 선로 여유용량**을 즉시 확인할 수 있게 만드는 기능.

**Why:** 태양광 사업자가 "어디에 설치 가능한가"를 빠르게 발굴하려면, 기존 지번/마을 기반 조회보다 "유효 타겟 시설 위에서 용량을 바로 보는" 흐름이 훨씬 효율적임. 의뢰자(2026-04-14 기준)가 직접 요청한 차기 기능.

**How to apply:** 아래 3개 레이어를 지도 위 토글 가능한 오버레이로 추가. 기존 `moveMapTo` / 지번 용량 팝업 로직을 재사용.

---

## 1. 유리온실 지도 레이어

- 데이터 소스: 농림부/국립농업과학원 등 공개 데이터 (유리온실 위치 포인트 또는 폴리곤). 출처 조사 필요.
- 지도에 유리온실 아이콘/폴리곤 표시
- 클릭 시 → 해당 위치 근처 선로 용량 팝업 (기존 지번 클릭 팝업 로직 재활용)

## 2. 축사 지도 레이어

- 데이터 소스: 축산물이력제 / 농식품부 축사 위치 데이터 등. 출처 조사 필요.
- 아이콘으로 지도에 표시, 클릭 시 근처 용량 팝업

## 3. 대형 건물 필터

- 설정에서 건축물 면적 기준 선택: **50평 / 100평 / 150평 / 200평 이상**
- 데이터 소스: 건축물대장 (국토부 공공데이터포털 API) — 동/지번 단위 건축물 면적 정보
- 조건 만족하는 건물을 지도에 표시, 클릭 시 근처 용량 팝업

---

## 기술 고려사항

- **데이터 양**: 전국 단위라 수십~수백만 포인트 가능성 → 줌 레벨 필터링 필수 (줌 작을 때는 클러스터만 / 가까울 때만 개별 표시)
- **근처 선로 용량 계산**: 이미 `geocode_cache` 에 지번 좌표 있고, `kepco_data` 에 시설별 용량 있음. "타겟 좌표 → 가까운 지번 top-K → 해당 지번 용량" 파이프라인.
- **레이어 토글**: 기존 사이드바 탭(주소검색/조건검색/변화추적)과 별도로 "타겟 검색" 탭 신설 고려
- **공통 로직 재사용**: 클릭 → 팝업은 [지적편집도 + 지도 클릭 팝업](project_cadastral_and_click_popup.md) 과 합쳐서 한 번에 구현하면 효율적

## 선행 작업

1. 각 데이터 소스 API/다운로드 방법 확보 (유리온실, 축사, 건축물대장)
2. 줌 레벨별 마커 최적화 ([project_zoom_level_optimization.md](project_zoom_level_optimization.md)) 먼저 또는 병행
3. 지도 클릭 → 지번 용량 팝업 ([project_cadastral_and_click_popup.md](project_cadastral_and_click_popup.md)) 먼저 (공통 팝업 로직)

## 우선순위

- 지적편집도+클릭팝업 → 줌 최적화 → 이 기능 순서가 자연스러움
- 3개 레이어 중 **축사/유리온실이 태양광 가장 유효** (옥상+유휴 공간 많음)
- 대형 건물 필터는 데이터 양 많아서 마지막
