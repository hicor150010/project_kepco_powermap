name: 모바일 개발 4대 원칙
description: 모바일 최적화 작업 시 반드시 준수해야 하는 핵심 원칙
type: feedback
---

모바일 작업 시 반드시 지켜야 할 4가지 원칙:

1. **웹(데스크톱)에 영향 없어야 함** — Tailwind `md:` 분기를 활용, 데스크톱 기존 동작/스타일 절대 변경 금지
2. **덕지덕지 코드 금지** — 깔끔하고 간결하게. 불필요한 wrapper, 중복 로직, 과도한 조건분기 NO
3. **크로스 브라우저 호환** — Safari/iPhone/다양한 기기에서 웹 모바일 버전과 동일하게. dvh, safe-area, input 16px 등 문서 확인 필수
4. **컴포넌트 재활용** — 새 컴포넌트 만들기 전에 기존 것 재활용. 단일 기능 단일 컴포넌트 원칙

**Why:** 유저가 "너무 덕지덕지", "단일 기능 단일 컴포넌트", "최대한 재활용" 을 여러 차례 강조. 웹 버전 깨뜨린 적 있어서 민감함.
**How to apply:** 모든 모바일 수정 시 md: 분기 확인, 기존 컴포넌트 재사용 가능한지 먼저 체크, 빌드 후 데스크톱 영향 없는지 검증.
