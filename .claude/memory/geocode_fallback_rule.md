type: project

KEPCO는 자체 주소 체계를 사용하며, "동" 하위에 "리"가 붙는 경우가 있다 (예: 가곡동 동산리).
카카오/VWorld 주소 API에는 이 조합이 존재하지 않아 지오코딩이 실패한다.

- "면+리" 조합은 정상 동작 (예: 서면 동산리 ✅)
- "동+리" 조합은 NOT FOUND (예: 가곡동 동산리 ❌)

**Why:** 카카오 주소 체계에서 "동"은 최하위 단위이고 그 아래 "리"는 없다. KEPCO만의 분류.

**How to apply:** 지오코딩 시 전체 주소 실패 → 리를 빼고 동만으로 재시도. 
주의: 리만으로 검색하면 다른 동의 동명 리가 나올 수 있어 위험하다 (서면 동산리 좌표가 가곡동 동산리로 매핑될 수 있음).

fallback 순서:
1. 전체 주소 (예: 전라남도 순천시 가곡동 동산리)
2. 실패 시 → 리 제거, 동까지만 (예: 전라남도 순천시 가곡동)

적용 파일: crawler/crawl_to_db.py, geocoder/geocode_worker.py
