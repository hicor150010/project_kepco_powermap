## 🎯 한 줄 정의

**지번 입력 → 영역 매핑 → 즉석 견적/수지분석 → 배치도/수지분석 PDF 출력**
영업 현장에서 태블릿으로 즉석 견적 + 고객 제출용 PDF 한 번에 생성하는 도구.

## 📌 의뢰자 핵심 원칙 (확정)

1. **매핑 영역 = 견적 기준** (필지 전체 ❌, 사용자가 그린 영역만 ⭕)
2. **시설 유형별 평수/단가 다름** — 4종 자동 보정 + 사용자 변경 가능
3. **단순 견적기 ❌** — 수지분석 + PDF 출력까지 풀 시스템
4. **일사량 데이터 별도 불필요** — `일평균 발전시간 4시간` 변수로 처리
5. **모든 변수 관리자 조정 가능**

## 📌 2차 스코프 확정 (의뢰자 동의)

- 건축물대장 정식 등록 시설만 대상 (비닐하우스·간이 슬레이트 등 가설건축물 제외)
- 평수 기준: **건축면적(`archArea`)만 사용** — 옥상 태양광 특성상 1층 바닥 면적이 곧 설치 가용 면적. `totArea` 사용 안 함.
- 의뢰자 추가 지시 (2026-04-19 카톡):
  - 유리온실/축사: **필터 없이 전국 전체 표시** (평수 구간 ❌)
  - 일반 건물(공장/창고): 평수 구간 7종 (50/100/150/200/300/400/500평↑)
- 2차 이름: 카톡엔 "**영업 대상 자동 발굴 시스템**" 으로 격상 (이전 "부가 정보 레이어" 라는 덤 인상 제거)

## 💵 합계 (카톡 안내본)

- 1차 660만 / 약 2~2.5개월 (조례 +30만 / +1주 포함)
- 2차 550만 / 약 2개월
- **합계 1,210만 / 약 4~4.5개월**
- 별도: 데이터 수집 인프라 1회성 20만 + 월 5만 (변동 가능 안내 포함)

## 💬 협상 내부 메모 — 조례 링크 옵션

의뢰자가 써니로직 화면 보내며 "시설별 견적할 때 조례 링크도 있어야 한다" 요청 (2026-04-19). 의뢰자 본인도 "이번 개발 범위 밖 아닌가요" 라고 인지.

- 정직한 견적: **80만 / 1주** (외부 API 신규 연동 + 시군구 매칭 + UI + 테스트)
- 실제 책정: **30만** — 사용자가 "추가 견적 요청이 약하다" 고 솔직히 인정, 마음 편한 가격으로 결정
- 손해 폭: 약 50만 (의도적 양보 — 1차 신뢰 구축 우선)
- **교훈**: 의뢰자가 "범위 밖이지 않나요" 라고 깔아준 상황은 정상 견적(80만)을 받기 좋은 자리였음. 다음 추가 요청 때는 정상 가격 권장.

조례 데이터: 법제처 자치법규 OpenAPI (`law.go.kr/DRF/lawService.do?target=ordin`). DB 저장 X — 조례 본문은 법제처가 관리 → 우리 측 유지보수 부담 0.

## 💵 인프라 책정 근거 (내부)

- GitHub Actions 자체는 무료 (Public repo) → 수집기 자체 비용 0
- 월 5만: Supabase Pro ($25 ≈ 3.5만) + 모니터링/운영 마진. 6~7개로 늘려도 동일 유지 가능 (간결한 숫자 의도)
- 1회성 20만: 수집기 +2개 셋업 + DB 인덱스 보강 + 모니터링 + 안정화
- 카톡 멘트: "서버 운영비는 사용자 수 증가, 신규 인프라 추가 시 변동될 수 있음" — Vercel 상위 요금제, AI 서버 등 추후 추가 가능성을 미리 안내
