docs/ 폴더 내 문서 (개발계획, 견적, PLAN, CRAWLING, SERVICES, SECRETS 제외, api_* 등) 수정은 사전 승인 요청 없이 바로 진행한다.

**Why:** 사용자가 2026-04-21 명시. "docs 문서 수정하는건 나한테 승인받지말고 그냥해". 코드 수정은 계획 승인 후 진행 원칙이지만, 문서는 사실 기록/정리 성격이라 왕복 비용이 낭비. 메모리 업데이트와 같은 결.

**How to apply:**
- 진행 상황 / 결제 / 완료 항목 / 진행률 등이 변할 때마다 관련 docs 즉시 반영
- 특히 [docs/개발계획.md](../../docs/개발계획.md) 의 Phase 상태, 완료 체크, 의뢰자 수락 시점 등
- [docs/견적_1차2차.md](../../docs/견적_1차2차.md) 의 완료 표시, 서비스 처리 반영
- 변경 후 사용자에게 "docs 어디어디 업데이트함" 한 줄 보고만
- 예외: `docs/SECRETS.local.md` 는 계속 사용자 주도 (자격증명 민감)
